# tone_verse — 中文版抑揚五步格寫詩工具

> 建立 2026-06-12。協助作家寫出符合聲調格律的中文詩。

## 格律規格（v0）

- 四句，每句十字。
- 奇數位（1,3,5,7,9）：「輕」位 — 工具可選**三聲 / 輕聲 / 兩者皆含**三種模式。
- 偶數位（2,4,6,8）：一聲。
- 第 10 字：四聲。

### 語言學備註（2026-06-12 已確認）
國語真輕聲幾乎都是語境性（的/了/嗎/子/頭/們⋯），辭典標輕聲讀音條目僅約一兩百個、多為助詞後綴，且輕聲不出現於句首。故「輕」位聲調做成可切換模式，純輕聲模式屬極限約束實驗。

## 功能規劃

1. **資料區**：全繁體中文國字 + 注音 + 聲調，可依聲調篩選（現無此類網站，為本案核心缺口）。
2. **寫詩區**：逐字格子，作家自選或手動輸入；即時驗證每格聲調約束。
3. **押韻候選**：根據已填內容推薦押韻字。
4. **語意排序**：候選字依與當前語境語意相近度排序。

## 資料來源候選

- 教育部《重編國語辭典修訂本》/《簡編本》open data（CSV，含注音）。
- g0v 萌典 moedict-data（GitHub，含異讀）。
- 注意：破音字（一字多音）需保留全部讀音，篩選時任一讀音符合即列入（標注）。

## 框架三問（2026-06-12 已答）

- **Q1 Done 標誌**：單檔 HTML，從空白 10 分鐘內寫完一首合律四句詩，每格有聲調過濾候選字。另需可切換的「純聲調查字」模式（無語意排序、無押韻），保留作家自主。
- **Q2 最容易死的方式**：Welcome 判定三天快速案、無死法。Claude 盯點＝字庫清洗 infrastructure trap，夠用即停。
- **Q3 留下的單一 artifact**：寫詩工具 HTML。不做自動校正、不做說明文件。
- **使用者**：作家（含 Welcome 自己）。

## 檔案

根目錄只留交付物，源檔全收進 `_src\`。

- `index.html` — **交付物**（根目錄，Vercel 部署入口）。單檔自包含工具（2.3MB，內嵌字庫＋字向量＋反義表）。直接雙擊開。**勿手動編輯**（巨檔不可維護），一律改 `_src\template.html` 再 rebuild。
- `_src\template.html` — HTML 模板（程式碼源頭，含 `/*__DB__*/` `/*__EMB__*/` `/*__ANTO__*/` `/*__WORDS__*/` 四注入點）。所有 UI／CSS／JS 改動改這支。
- `_src\generate_db.py` — 字庫管線（wordfreq 頻 → Big5＋OpenCC 濾繁 → pypinyin 注音 → 解析調/聲母/韻）。
- `_src\gen_embeddings.py` — 語意字向量管線（model2vec 靜態向量，純 numpy 不需 torch → int8 量化）。
- `_src\gen_words.py` — 詞庫管線（wordfreq 詞頻 → OpenCC 濾繁 → 只留「每字都在單字庫」的多字詞）。
- `_src\chars_db.json` — 聲調字庫（5855 字，主讀音，含 c/f/z/t/sm/y）。
- `_src\embeddings.json` — int8 量化字向量（256d × 5855，base64，~1.9MB）。
- `_src\antonyms.json` — 手工反義詞對表（~120 對）。
- `_src\words.json` — 詞庫（46431 詞，純詞字串陣列依詞頻排序；聲調/注音/韻/聲母/向量全部 runtime 從單字庫推，故不另存）。

### 重新生成指令（從專案根目錄執行）
```
cd _src; python generate_db.py; python gen_embeddings.py; python gen_words.py; cd ..   # 重建字庫/字向量/詞庫（首次會下載 model2vec 多語模型；gen_words 須先有 chars_db.json）
python -c "t=open('_src/template.html',encoding='utf-8').read();t=t.replace('/*__DB__*/',open('_src/chars_db.json',encoding='utf-8').read());t=t.replace('/*__EMB__*/',open('_src/embeddings.json',encoding='utf-8').read());t=t.replace('/*__ANTO__*/',open('_src/antonyms.json',encoding='utf-8').read());t=t.replace('/*__WORDS__*/',open('_src/words.json',encoding='utf-8').read());open('index.html','w',encoding='utf-8').write(t)"
```
> 多數 UI 改動只需改 `_src\template.html` ＋跑上面最後一行（注入指令），不必重建字庫。

依賴：`pip install pypinyin wordfreq jieba opencc-python-reimplemented model2vec`

## 已知限制（v3 backlog）

- **破音字**：採主讀音，每字一調。長/重/行等多音字僅出現於主讀音之聲調。
- **輕聲池僅 18 字**（全為助詞，且不出現於句首）→ 純輕聲格律實質不可行，三聲模式才實用。
- **押韻按完整韻母**（含介音），未做介音不敏感的寬韻分組。
- **語意品質中上**：單字 model2vec 靜態向量，愛-戀0.86、愛-石0.06 區分清楚；山-水偏低。要更懂詩意需接 LLM。
- **反義表約 120 對**，覆蓋常見反義；罕見反義無標記。
- **詞語模式**：詞的聲調沿用單字主讀音（破音字僅主讀音），故某些詞的真實連讀聲調可能與標示有出入；不做變調（如三三連讀變調）。詞庫只收「每字都在 5855 單字庫」的詞，故含罕字的詞不會出現。詞語模式忽略上方「聲調」勾選列（改由格律逐位決定）。
